## 2. رؤية المشروع

**الاسم:** وضوح | Wuduh  
**الشعار المقترح:** "إدارة مشاريعك باحترافية PMP — بدون مدير مشاريع"

**المشكلة:**  
الشركات الصغيرة والمتوسطة لا تستطيع تحمّل تكلفة مدير مشاريع PMP (15,000+ ريال/شهر) لكنها تحتاج إدارة مشاريع احترافية.

**الحل:**  
منصة AI متخصصة في إدارة المشاريع وفق معايير PMI/PMBOK تبني خطة مشروع كاملة في دقائق، وتولّد جميع المخرجات المطلوبة جاهزة للاستخدام.

**السوق الأول:** شركات البناء والإنشاءات في المملكة العربية السعودية  
**التوسع لاحقاً:** قطاع التقنية، ثم عالمياً

## 4. مسار المستخدم الكامل (User Journey)

```
[1] التسجيل
    ↓
[2] تحديد نوع المستخدم (فرد / شركة صغيرة / مؤسسة)
    ↓
[3] أسئلة عن المجال (البناء والإنشاءات في البداية)
    ├── نوع المشاريع التي تنفذها
    ├── حجم فريقك
    ├── مدة مشاريعك المعتادة
    └── أكبر تحدياتك في الإدارة
    ↓
[4] إنشاء مشروع جديد
    ├── اسم المشروع
    ├── وصف المشروع
    ├── تاريخ البدء والانتهاء
    ├── الميزانية الكلية
    ├── عدد أعضاء الفريق
    ├── الجهة / العميل
    └── أي متطلبات خاصة
    ↓
[5] AI يحلل المعلومات ويبني الأجندة الكاملة
    ├── نظرة عامة على المشروع
    ├── المراحل الرئيسية (Phases)
    ├── المهام والمسؤوليات
    ├── الجدول الزمني المقترح
    ├── المخاطر المتوقعة
    └── مؤشرات النجاح (KPIs)
    ↓
[6] المستخدم يراجع الأجندة
    ├── يعتمدها كما هي
    ├── يعدّل عليها
    └── يضيف ملاحظاته
    ↓
[7] اختيار المخرجات المطلوبة
    (التفصيل في القسم 5)
    ↓
[8] AI يولّد المخرجات كاملة
    ↓
[9] تحميل / مشاركة / متابعة
```

- **Stakeholder Register** — سجل أصحاب المصلحة وخطة إدارتهم

- **Stakeholder Presentation** — عرض لأصحاب المصلحة

## 6. التقنيات المستخدمة (Tech Stack)

### الواجهة الأمامية (Frontend)
- **Framework:** Next.js 14 (React)
- **Styling:** Tailwind CSS
- **Components:** shadcn/ui
- **Charts:** Recharts / Chart.js
- **Language:** TypeScript

### الخلفية (Backend)
- **Framework:** Next.js API Routes (Server Actions)
- **Auth:** Clerk (مجاني حتى 10,000 مستخدم)
- **Database:** Supabase (PostgreSQL) — مجاني حتى 500MB
- **Storage:** Supabase Storage (للملفات)
- **Email:** Resend (مجاني حتى 3,000 بريد/شهر)

### الذكاء الاصطناعي
- **Primary AI:** Claude API (Anthropic) — claude-sonnet-4-6
- **Approach:** Prompt Engineering متخصص بـ PMBOK Guide 7th Edition
- **Context:** PMBOK standards كـ system prompt ثابت
- **Validation:** كل مخرج يُتحقق منه ضد معايير PMI قبل التسليم

### توليد المخرجات
- **PDF:** PDFKit أو Puppeteer
- **Excel / XLSX:** ExcelJS
- **PowerPoint / PPTX:** PptxGenJS
- **Word / DOCX:** docx library

### الدفع
- **Primary:** Stripe (عالمي — بطاقات، Apple Pay، Google Pay)
- **Local (لاحقاً):** Moyasar (السوق السعودي)

### الاستضافة
- **Hosting:** Vercel (مجاني في البداية)
- **Domain:** مؤجل حتى قبل الإطلاق بأسبوع

## 13. هوية الشركة والعلامة التجارية (Company Branding)

**الخطط:** احترافي ($79) + مؤسسي ($199)  
**الأولوية:** عالية — تميّز تنافسي مباشر  
**المرحلة:** Q3 2026

### الوصف
يتيح وضوح للشركات رفع هويتها التجارية الكاملة في لوحة التحكم، وتُحقن تلقائياً في جميع الوثائق المولّدة بالذكاء الاصطناعي.

### ما يمكن تخصيصه

| الحقل | التفاصيل | متاح في |
|---|---|---|
| شعار الشركة (Logo) | PNG/SVG يظهر في كل وثيقة | احترافي + مؤسسي |
| الترويسة الرسمية (Letterhead) | قالب ورق رسمي للـ PDF و Word | احترافي + مؤسسي |
| الألوان الرئيسية | Primary + Secondary — تنعكس على PPTX والجداول | احترافي + مؤسسي |
| معلومات المنشأة | اسم، عنوان، هاتف، بريد، موقع، رقم تجاري | احترافي + مؤسسي |
| هوية متعددة الأقسام | كل قسم له هويته المستقلة | مؤسسي فقط |

### كيف تظهر في الوثائق

- **Word (.docx):** شعار في رأس الصفحة + ترويسة كخلفية + معلومات الشركة في الغلاف والفوتر
- **Excel (.xlsx):** شعار في خلية الرأس + ألوان الشركة للجداول والرسوم البيانية
- **PowerPoint (.pptx):** شعار في كل شريحة + ألوان الشركة للعناوين والخلفيات + معلومات في شريحة الغلاف
- **PDF:** ترويسة + تذييل بمعلومات الشركة + شعار في كل صفحة

### ملاحظة مهمة
إضافة معلومات المنشأة للوثائق **اختيارية** — يتحكم المستخدم بتفعيلها من إعدادات كل مشروع على حدة.

### جداول قاعدة البيانات المطلوبة

```sql
CREATE TABLE company_profiles (
  id              UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(clerk_id) ON DELETE CASCADE,
  company_name    TEXT,
  logo_url        TEXT,              -- Supabase Storage URL
  logo_path       TEXT,             -- internal storage path
  letterhead_url  TEXT,
  letterhead_path TEXT,
  primary_color   TEXT DEFAULT '#1B4FD8',
  secondary_color TEXT DEFAULT '#D4A017',
  address         TEXT,
  phone           TEXT,
  email           TEXT,
  website         TEXT,
  cr_number       TEXT,             -- رقم السجل التجاري (اختياري)
  include_in_docs BOOLEAN DEFAULT TRUE,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);

-- RLS
ALTER TABLE company_profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_own_company_profile" ON company_profiles
  FOR ALL USING (user_id = current_setting('app.user_id', true));
```

### صفحات الواجهة المطلوبة
- `/settings/branding` — إعدادات هوية الشركة (رفع شعار + ترويسة + ألوان + معلومات)
- تعديل `src/app/api/deliverables/generate/route.ts` لحقن بيانات الشركة في prompt الذكاء الاصطناعي

## 14. نماذج الوثائق المحفوظة (Saved Document Templates)

**الخطط:** احترافي ($79) + مؤسسي ($199)  
**الأولوية:** عالية — تسريع سير العمل وتوحيد التوثيق  
**المرحلة:** Q3 2026

### الوصف
قبل توليد الوثائق، يختار المستخدم نموذجاً محفوظاً (قالب). الذكاء الاصطناعي يبني كل الوثائق بناءً على النموذج المختار + بيانات المشروع + هوية الشركة.

### مسار المستخدم

```
اعتماد الأجندة
    ↓
توليد الوثائق ← يظهر منتقي النماذج
    ↓
اختيار نموذج (نماذج وضوح / نماذج الشركة / نماذج المؤسسة)
    ↓
الذكاء الاصطناعي يبني بناءً على: النموذج + بيانات المشروع + هوية الشركة
    ↓
25+ وثيقة جاهزة بتصميم موحّد
```

### أنواع النماذج

| النوع | الوصف | من يديرها |
|---|---|---|
| نماذج وضوح الافتراضية | نماذج احترافية جاهزة لكل قطاع من الـ 23 | وضوح (read-only) |
| نماذج الشركة المخصصة | المستخدم يُنشئ نموذجه الخاص + توجيهات للذكاء الاصطناعي | المستخدم |
| نماذج المؤسسة المشتركة | مدير الحساب يُشاركها مع كل الأقسام | Enterprise admin |

### جداول قاعدة البيانات المطلوبة

```sql
CREATE TABLE document_templates (
  id              UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  user_id         TEXT,              -- NULL = وضوح default templates
  org_id          UUID,              -- للنماذج المؤسسية المشتركة
  name            TEXT NOT NULL,
  description     TEXT,
  industry        TEXT,              -- قطاع مخصص أو NULL = عام
  prompt_context  TEXT,             -- توجيهات إضافية للذكاء الاصطناعي
  structure       JSONB,            -- هيكل النموذج (sections, tone, length)
  is_default      BOOLEAN DEFAULT FALSE,
  is_shared       BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### صفحات الواجهة المطلوبة
- `/settings/templates` — إدارة النماذج (إنشاء / تعديل / حذف)
- تعديل `/projects/[id]/deliverables` — إضافة Template Picker قبل زر التوليد
- تعديل `src/app/api/deliverables/generate/route.ts` — تمرير `templateId` وحقن `prompt_context` في الـ prompt
